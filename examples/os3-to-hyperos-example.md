# 示例：从 OS3 老规范迁移到 HyperOS 规范

## 输入示例

```text
OS3 老规范：
列表项按下时背景变深，右侧箭头表示可进入下一级页面。禁用项文字置灰，不响应点击。

新场景：
HyperOS 设置页中的可点击设置项，需要支持深色模式、长标题和大屏设备。
```

## 输出片段示例

```markdown
## 状态与变体

| 状态 | 触发条件 | 视觉变化 | 行为规则 |
| --- | --- | --- | --- |
| 默认 | 设置项可操作 | 使用默认表面色与主文本层级 | 点击进入下一级页面 |
| 按下 | 用户按下设置项 | 容器呈现即时按压反馈 | 释放后触发跳转 |
| 禁用 | 当前设置不可操作 | 文本和图标降低强调层级 | 不响应点击，也不进入焦点顺序 |

## 从 OS3 迁移

| OS3 规则 | HyperOS 处理 | 原因 |
| --- | --- | --- |
| 按下时背景变深 | 调整为使用 HyperOS 状态反馈 token | 适配深色模式和统一状态层级 |
| 右侧箭头表示下一级 | 保留 | 仍然是可导航设置项的有效 affordance |
| 禁用项文字置灰 | 调整 | 不直接指定灰色值，改用禁用态 token |
```
