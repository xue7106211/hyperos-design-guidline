# HyperOS 组件原则参考

当为具体 HyperOS 组件、设计模式或基础设计主题撰写 `核心原则` 或原则相关章节时使用本参考。只选择当前规范需要的条目，不要把整份参考复制进正文。

来源：根据用户提供的组件原则板图片整理。`待补充` 条目表示仍需用户确认或补充来源材料。

## 使用规则

- 撰写 `核心原则` 前，先将目标组件匹配到本参考中最接近的条目。
- 若组件有匹配条目，应按原词引用原则；仅在必要时补充简短的“规范落点”解释。
- 如果一个规范涉及多个组件，只引入当前规范需要的条目。
- 如果目标组件未列出，可使用所属分类原则作为兜底，并将组件专属原则标为 `待确认`。
- 不要为了适配文档结构而改名、合并或重写原则。
