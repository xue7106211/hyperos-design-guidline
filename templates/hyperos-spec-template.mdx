## 设计原则

| 原则 | 说明 |
| --- | --- |
| 稳定优先 | 保持关键操作位置和反馈一致，降低跨场景迁移成本。 |
| 反馈明确 | 用户操作后应获得及时、可理解的状态反馈。 |
| 场景适配 | 根据设备、密度、主题和输入方式调整呈现与交互。 |

## 结构与组成

| 元素 | 必需 | 说明 |
| --- | --- | --- |
| 容器 | 是 | 承载主体内容和交互热区。 |
| 主内容 | 是 | 展示用户需要识别或操作的核心信息。 |
| 辅助内容 | 否 | 展示补充说明、状态或次级操作。 |

## 状态与变体

| 状态 | 触发条件 | 视觉变化 | 行为规则 |
| --- | --- | --- | --- |
| 默认 | 初始可用 | 使用默认容器与文字层级 | 可点击或可聚焦 |
| 按下 | 用户按下但未释放 | 呈现即时反馈 | 释放后触发操作 |
| 禁用 | 当前不可操作 | 降低强调层级 | 不响应点击 |

## 交互行为

| 行为 | 规则 |
| --- | --- |
| 点击 | 命中热区后触发主操作，并给出即时反馈。 |
| 长按 | 仅在场景需要二级操作时启用。 |
| 键盘 / 焦点 | 支持可聚焦元素的顺序导航与可见焦点反馈。 |

## 内容与文案

- 文案应简洁、明确，避免仅依赖颜色表达状态。
- 动态内容应定义最大长度、截断方式和空值策略。

| 背景 | `{color.surface.*}` | 根据层级和主题选择 |

## 动效规范

| 动效 | 触发 | 规则 |
| --- | --- | --- |
| 状态反馈 | 点击、按下、选中 | 使用轻量过渡，避免影响操作效率。 |

## 无障碍与可用性

- 可操作区域应满足当前平台的最小触控目标要求。
- 关键状态不应仅依赖颜色区分。
- 深色模式和浅色模式均应保持足够对比度。
