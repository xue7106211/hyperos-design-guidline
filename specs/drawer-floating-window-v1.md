# 抽屉浮窗 Drawer Floating Window V1

## 概述

抽屉浮窗是一种从屏幕边缘进入、覆盖或并列于当前页面之上的容器型界面，用于承载临时任务、轻量页面、辅助配置或与背景内容并行的操作。它在手机竖屏场景下通常表现为底部抽屉；在大屏、折叠屏、平板或横屏场景下，可根据断点与业务任务转化为浮窗。

本规范基于旧版「抽屉窗口 Drawer Window」与「半屏面板」资料整理，先定义 V1 可执行规则，覆盖结构、分类、交互、适配、视觉与迁移约束。V1 用于设计、研发、QA 先对齐方向与边界；精确 token、断点、动效曲线与平台 API 仍需接入 HyperOS 设计系统后补齐。

## 适用范围

| 范围 | 说明 |
| --- | --- |
| 适用 | HyperOS 应用内从边缘弹出的模态抽屉、非模态抽屉，以及在大屏场景由抽屉转化而来的浮窗。 |
| 主要设备 | 手机竖屏、手机横屏、折叠屏、平板、分屏窗口。 |
| 主要场景 | 临时任务、轻量页面、新建/编辑流程、选择器、系统能力调用、预览同时编辑或配置。 |
| 不适用 | 全屏主页面、Toast、Snackbar、普通 Dialog、底部导航、常驻侧边栏。 |
| 依赖 | 标题栏、浮窗、遮罩、系统返回、手势导航、窗口断点与 HyperOS token。 |

## 设计原则

| 原则 | 说明 |
| --- | --- |
| 任务轻量化 | 使用抽屉浮窗承接不必离开当前上下文的任务，减少全屏跳转带来的割裂感。 |
| 背景关系明确 | 模态抽屉应阻断背景操作；非模态抽屉应允许背景继续预览或操作，不应混淆两类关系。 |
| 档位可预期 | 高度变化应在有限档位内完成，默认、最大、最小高度应由任务类型清晰定义。 |
| 手势不争抢 | 内容滚动、抽屉拖拽与关闭手势应有明确优先级，避免用户无法判断当前操作对象。 |
| 大屏自然转化 | 当屏幕宽度或窗口形态不适合底部抽屉时，应转化为浮窗或分栏式浮层，而不是强行拉伸半屏形态。 |

## 结构与组成

| 元素 | 必需 | 规则 |
| --- | --- | --- |
| 容器 | 是 | 承载标题区与内容区；应具备独立层级、圆角、背景、阴影或等效分层表现。 |
| 遮罩 | 视类型而定 | 模态抽屉应提供遮罩并阻断背景交互；非模态抽屉不应提供遮罩。 |
| 滑动把手 | 可选 | 用于提示可拖拽、切换高度或关闭；当仅有一个固定档位且不可关闭时，可不展示。 |
| 标题区 | 可选 | 可包含标题、关闭、取消、确认或其他页面级操作；规则应继承标题栏，但不支持大字标题。 |
| 内容区 | 是 | 展示核心内容、表单、列表、宫格、选择器或配置项；底部按钮栏属于内容区的一部分。 |
| 操作区 | 可选 | 当存在主操作、确认、取消等底部操作时，应与内容滚动区域保持清晰边界。 |

## 类型与使用场景

| 类型 | 背景关系 | 默认用途 | 典型场景 | 关键约束 |
| --- | --- | --- | --- | --- |
| 模态抽屉窗口 | 背景不可交互 | 临时展示少量信息或操作项 | 操作菜单、少量配置、确认前选择 | 应有遮罩；背景内容不可操作；关闭后返回发起页面。 |
| 模态抽屉页面 | 背景不可交互 | 以更轻量方式承载原本的临时全屏页 | 新建闹钟、编辑条目、跨进程打开轻量页、搜索选择 | 可展开到较大档位；系统返回应退出抽屉页面。 |
| 非模态抽屉 | 背景可交互 | 与背景内容并行预览、编辑或配置 | 日历日程联动、小部件编辑、AOD 预览配置 | 不应有遮罩；背景可继续操作；抽屉高度变化可影响背景内容布局。 |
| 大屏浮窗 | 视任务而定 | 抽屉在大屏或横屏下的转化形态 | 平板、折叠屏、横屏、宽窗口 | 不应简单铺满底部；应按浮窗规则控制宽度、位置与层级。 |

## 布局规则

### 宽度与位置

| 场景 | 规则 |
| --- | --- |
| 手机竖屏 | 默认从底部弹出，宽度贴合当前窗口宽度。 |
| 手机横屏 | V1 不建议继续使用全宽底部抽屉；应优先评估转化为浮窗。若业务暂不支持，应明确强制竖屏或降级策略。 |
| 折叠屏 / 平板 | 模态抽屉窗口与模态抽屉页面建议转化为浮窗；非模态抽屉可继续以半屏/分栏形态服务并行任务。 |
| 左右分屏 | 每个应用分屏区域可独立使用抽屉浮窗；不应跨越当前应用窗口边界。 |
| 上下分屏 | 抽屉高度应基于当前应用窗口高度计算，而不是物理屏幕高度。 |

### 高度档位

| 档位 | 规则 | 适用 |
| --- | --- | --- |
| 收起态 | 高度接近标题区或最小可识别高度；仅在业务需要保留入口时使用。 | 非模态抽屉、少数可持续存在的工具面板。 |
| M 档 | 中等高度，旧资料中以屏幕高度约 70% 作为参考；最终比例待设计系统确认。 | 多数默认半屏任务。 |
| L 档 | 接近全屏，需保留状态栏、安全区与顶部间距；不应伪装为完整全屏页。 | 抽屉页面、复杂内容、长列表、表单编辑。 |
| 自适应高度 | 内容位于 S-M 区间时，可按内容高度自适应；最低高度待确认。 | 少量内容的模态抽屉窗口。 |
| 自定义档位 | 由业务定义默认高度、最大高度、最小高度与切换逻辑。 | 非模态抽屉、需要背景预览的编辑工具。 |

高度变化应遵循有限档位，不支持无极悬停。若内容因大字号或多语言超过 M 档容量，建议直接进入 L 档。

### 标题区

- 标题区可左对齐或居中，需与所属类型保持一致。
- 仅提供「关闭」或「取消」时，操作可位于左侧。
- 抽屉页面承载原全屏页时，标题栏建议使用导航吸顶样式。
- 不支持大字标题；标题长文本应按标题栏规则截断、换行或降级展示。

## 交互行为

### 打开与关闭

| 行为 | 模态抽屉 | 非模态抽屉 |
| --- | --- | --- |
| 打开 | 从发起控件或底部边缘进入，背景进入遮罩与不可交互状态。 | 从底部边缘进入，背景保持可见且可交互。 |
| 关闭按钮 | 可提供关闭、取消或返回入口。 | 若抽屉中途被拉出或可被移除，应提供关闭入口；若抽屉始终存在且不可关闭，可不提供。 |
| 点击遮罩 | V1 建议不以点击遮罩作为默认关闭方式；是否允许点击关闭需按业务风险确认。 | 不存在遮罩。 |
| 系统返回 | 应关闭抽屉并回到发起页面；若抽屉内部存在页面栈，应优先回退内部页面。 | 应按业务定义：关闭抽屉、回退内部页面或返回背景页面，需避免不可预期。 |
| 手势关闭 | 在最小档位继续下滑可关闭；高风险任务应要求显式确认或保存提示。 | 可关闭场景遵循同样规则；不可关闭场景不应响应关闭手势。 |

### 档位切换

| 操作区域 | 上滑规则 | 下滑规则 |
| --- | --- | --- |
| 标题区 / 把手 | 未到最大档位时，短上滑切换到临近更大档位，长上滑切换到最大档位；已到最大档位时仅轻微回弹。 | 未到最小档位时，短下滑切换到临近更小档位，长下滑可关闭；已到最小档位时继续下滑可关闭。 |
| 内容区顶部或无滚动容器 | 优先扩展抽屉档位；无更大档位时再滚动内容或触发回弹。 | 优先收缩抽屉档位；无更小档位时关闭抽屉。 |
| 内容区中间 | 优先滚动内容，不应抢占为抽屉高度变化。 | 优先滚动内容回到顶部；到顶部后再触发抽屉收缩或关闭。 |

### 页面跳转

- 抽屉内容区可承载内部页面跳转，但抽屉框架、遮罩关系与把手规则应保持稳定。
- 进入二级内容后，系统返回应优先回退抽屉内部层级；无内部层级时再关闭抽屉。
- 不建议在抽屉内嵌套新的模态抽屉；如必须使用，应明确层级、返回顺序与遮罩叠加规则。

## 状态与变体

| 状态 | 触发条件 | 视觉变化 | 行为规则 |
| --- | --- | --- | --- |
| 默认 | 抽屉已打开且可交互 | 容器、标题区、内容区按默认层级展示 | 用户可操作内容或切换档位。 |
| 拖拽中 | 用户拖动把手、标题区或内容区顶部 | 容器跟随手势移动；背景遮罩或并行区域可按进度变化 | 释放后吸附到目标档位或关闭。 |
| 最大档位 | 抽屉已到 L 档或业务最大高度 | 顶部仍保留抽屉形态与安全间距 | 继续上滑仅回弹，不应进入无定义高度。 |
| 最小档位 | 抽屉已到收起态或业务最小高度 | 展示最小可识别内容或入口 | 继续下滑可关闭，除非业务定义不可关闭。 |
| 加载中 | 内容尚未完成加载 | 内容区展示加载反馈，框架保持稳定 | 不应因加载完成造成突兀档位跳变。 |
| 空状态 | 内容为空或暂无结果 | 内容区展示空状态说明与可选行动 | 不应关闭抽屉代替空状态说明。 |
| 错误状态 | 加载失败或权限不足 | 内容区展示错误信息与重试/引导 | 不应让用户停留在无反馈空白容器。 |
| 禁用关闭 | 任务必须完成或不可中断 | 关闭入口隐藏或禁用，并提供原因或替代操作 | 应谨慎使用，避免造成困住用户。 |

## 内容与文案

- 标题应说明当前任务，而不是重复发起按钮文案。
- 关闭、取消、完成、保存等操作文案应与任务结果一致；存在未保存内容时应明确提示。
- 空状态、错误状态和权限状态应放在内容区内处理，不应改变抽屉类型。
- 多语言长标题应遵从标题栏适配规则；业务不应依赖固定字数保证布局。

## 视觉规范

| 属性 | V1 规则 | Token 状态 |
| --- | --- | --- |
| 容器背景 | 应与当前页面背景形成清晰层级，并支持浅色/深色模式。 | 待接入 `{color.surface.*}`。 |
| 遮罩 | 模态抽屉使用遮罩降低背景强调层级；非模态抽屉不使用遮罩。 | 待接入 `{color.overlay.*}`。 |
| 圆角 | 顶部边缘应体现浮层容器形态；大屏浮窗需四角圆角。 | 待接入 `{radius.*}`。 |
| 阴影 / 海拔 | 应表达抽屉高于页面内容；非模态抽屉尤其需要清晰区分 Z 轴层级。 | 待接入 `{elevation.*}`。 |
| 标题文字 | 使用标题栏对应文字层级，不支持大字标题。 | 待接入 `{font.title.*}`。 |
| 把手 | 低强调、可识别，不应与标题或主操作竞争。 | 待接入 `{color.icon.tertiary}`。 |

## 动效规范

- 打开与关闭应使用上下方向的进入/退出动效，来源方向与抽屉位置一致。
- 档位切换应吸附到明确高度，并给出自然回弹；不应停留在未定义的中间高度。
- 模态抽屉打开时，遮罩透明度应与容器进入同步变化。
- 非模态抽屉调整高度时，背景内容若需要避让或重排，应与抽屉运动协调完成。
- 具体时长、曲线和弹性参数待接入 HyperOS motion token。

## 无障碍与可用性

- 模态抽屉打开后，焦点应限制在抽屉内部；关闭后焦点应回到发起控件。
- 非模态抽屉不应阻断背景焦点，但焦点顺序需在背景与抽屉之间可预测。
- 关闭、取消、完成等操作必须具备可读标签，不应只依赖图标。
- 把手若支持拖拽，应提供等效的键盘、开关控制或明确的可访问替代操作。
- 文本最大放大倍数参考旧资料为 1.45x；最终应与 HyperOS 字号无障碍策略确认。
- 图标通常不随文本等比例放大，需保证触控热区仍满足平台最小要求。

## 适配与响应式

| 场景 | 规则 |
| --- | --- |
| 浅色 / 深色模式 | 容器、遮罩、文字、分割、阴影应使用语义 token，不应写死颜色值。 |
| 大字号 | 标题区遵从标题栏规则；内容区遵从内部控件规则；高度不足时升级到更大档位。 |
| 多语言 | 标题与操作文案应支持截断或换行策略；底部操作不应因长文案溢出。 |
| 分屏应用 | 每个分屏窗口独立计算抽屉宽高与安全区；非模态抽屉可与每屏内容并列使用。 |
| 平板 / 折叠屏 | 模态类抽屉优先转化为浮窗；非模态类抽屉按并行任务关系决定是否保留半屏形态。 |

## 边界场景

| 场景 | 处理规则 |
| --- | --- |
| 内容过长 | 内容区应滚动；当超过 M 档容量时使用 L 档，不应无限增高。 |
| 内容过少 | 可使用自适应高度，但不得低于最小可操作高度。 |
| 未保存更改 | 关闭前应提示保存、放弃或继续编辑。 |
| 键盘弹出 | 抽屉应避让键盘，保证当前输入项与主操作可见。 |
| 内部滚动嵌套 | 应定义主滚动容器，避免多层纵向滚动导致手势冲突。 |
| 背景可交互但被遮挡 | 非模态抽屉应让背景内容根据抽屉高度调整布局或明确被覆盖区域不可见。 |
| 多窗口叠加 | 不建议同时打开多个抽屉；若有系统弹窗叠加，应明确最高层级与返回顺序。 |

## 实现说明

| 项目 | 说明 |
| --- | --- |
| 类型声明 | 组件应显式声明 `modal` / `nonModal` / `floating`，不应仅由是否展示遮罩反推类型。 |
| 档位配置 | 业务应提供默认高度、最大高度、最小高度、是否可关闭、是否可拖拽等配置。 |
| 手势仲裁 | 实现需区分标题区拖拽、内容区滚动、内容区顶部下拉和系统返回手势。 |
| 内部导航 | 抽屉页面如承载页面栈，应维护独立返回栈，并与系统返回顺序一致。 |
| 状态保存 | 档位变化、未保存输入、加载状态不应因旋转、分屏变化或窗口尺寸变化丢失。 |
| 大屏转化 | 同一业务入口在不同断点下可映射为抽屉或浮窗，但任务语义与返回规则应保持一致。 |

## QA 检查清单

- [ ] 模态抽屉打开后，背景不可点击、不可聚焦，系统返回可关闭或回退内部页面。
- [ ] 非模态抽屉打开后，背景仍可按预期预览或操作，且无遮罩。
- [ ] 标题区、内容区顶部、内容区中间的上滑/下滑优先级符合规范。
- [ ] M/L/收起态/自定义档位能正确吸附，不停留在未定义高度。
- [ ] 长内容、大字号、多语言、空状态、错误状态不会破坏布局。
- [ ] 键盘弹出时，输入项和主操作仍可见且可操作。
- [ ] 手机竖屏、横屏、折叠屏、平板、分屏下的形态转化符合业务定义。
- [ ] 深色模式与浅色模式下，容器层级、遮罩和文字可读性一致。

## 从旧规范迁移

| 旧规范规则 | V1 处理 | 原因 |
| --- | --- | --- |
| 模态抽屉用于临时任务，背景有遮罩且不可交互 | 保留 | 仍是区分临时任务与并行任务的核心规则。 |
| 非模态抽屉用于并行任务，背景可交互 | 保留并强化 | HyperOS 多任务、大屏与预览编辑场景更依赖此规则。 |
| 抽屉高度遵循低、中、高三个尺寸，不支持无极悬停 | 调整 | V1 改写为收起态、M、L、自适应与业务自定义档位，但仍强调有限档位吸附。 |
| 横屏暂不支持，强制竖屏 | 调整 | V1 不直接规定强制竖屏，而是建议横屏优先转化为浮窗或提供明确降级。 |
| 折叠屏、PAD 默认为浮窗，小部件特殊适配 | 保留并泛化 | V1 将其扩展为大屏转化规则，允许非模态抽屉按并行任务保留半屏形态。 |
| 标题栏规则与全屏页面一致，但不支持大字标题 | 保留 | 维持页面级导航一致性，同时避免抽屉标题区占用过高。 |
| 非模态单档且不可关闭时可隐藏把手 | 保留 | 避免把手暗示不存在的拖拽或关闭能力。 |
| 点击遮罩不可关闭窗口 | 待确认 | 半屏面板资料中存在“点击可退出半屏”的相反描述，需按 HyperOS 模态策略统一。 |
