# 列表（List）
## 一、概述
列表是 HyperOS 中用于连续呈现同类条目的界面模式。它帮助用户在设置、消息、媒体、联系人等场景中快速浏览、定位并操作一组结构相近的信息行。
列表适合条目数量可变、需要纵向滚动浏览、且单行（或固定多行）承载主信息与附属操作的任务。不适合用卡片瀑布流表达的强视觉浏览，也不适合需要复杂二维对比的数据表。
- **识别性**：单行主信息应一眼可辨；左侧标识（图标、头像、缩略图）与主文本共同建立条目身份，避免装饰盖过语义。
- **可扫描性**：同一分组内对齐、间距与信息位保持一致，使用户可纵向扫视而非逐字阅读。
- **层次性**：主文本、辅助文本、元数据与右侧附属信息分层表达；分组与子标题用于分隔不同信息簇。
- **可操作性**：可点、可选、可滑出的操作边界清晰；状态反馈能让用户确认当前行是否可交互、是否已选中。
## 二、类型与结构
| 类型 | 用途 | 典型场景 | 关键约束 |
| --- | --- | --- | --- |
| 效率型列表 | 以文本与轻量图标完成查找与跳转 | 设置项、开关项、通讯录、文件名列表 | 控制装饰密度；优先单列、固定信息位 |
| 内容型列表 | 文本与媒体混排，强调内容预览 | 新闻摘要、相册条目、播客单集、下载任务 | 行高随媒体占比提升；同组内布局应一致 |
| 选择型列表 | 单选/多选集合中的条目 | 批量管理、筛选条件、分享对象 | 选中态必须可区分；禁用项不可进入选择序列 |
| 操作扩展列表 | 主行为在行内，次要行为通过侧滑等扩展露出 | 消息删除/收藏、待办完成、会话置顶 | 扩展操作数量应克制；图标表意须明确 |
类型选择：查找与配置优先效率型；依赖封面/缩略图用内容型；批量确认用选择型；低频或破坏性次要操作放入操作扩展，整行点击仍承担主任务。
结构组成：
| 元素 | 必需 | 说明 |
| --- | --- | --- |
| 列表容器 | 是 | 承载滚动、分组与条目间距；同屏内同类列表视觉规范应一致 |
| 列表项 | 是 | 等高或按内容档位分行的单条；整行通常为主要热区 |
| 左侧标识区 | 否 | Badge、系统图标、头像、应用图标或预览图；用于识别性 |
| 主内容区 | 是 | 主文本必选；可含辅助文本、描述、时间戳等次级信息 |
| 右侧附属区 | 否 | 值文本、开关入口箭头、功能图标，或图标+文本组合 |
| 分组标题 | 否 | 对条目分簇；标题本身一般不可作为业务主操作 |
| 分割与间距 | 是 | 组内节奏统一；不同结构的行不应混在同一视觉分组中强行对齐 |
信息位：主内容区承载语义核心；右侧附属区承载状态值或进入下一步的暗示。长文本优先「主标题 + 次级说明」。效率型以纯文本或轻量图标为主；内容型媒体与文本混排且同组比例统一；侧滑操作变体在横向手势后露出有限次要操作，同产品内露出语言应统一。
## 三、交互与状态
| 状态 | 触发条件 | 视觉变化 | 行为规则 |
| --- | --- | --- | --- |
| 默认 | 条目可用且未聚焦 | 使用默认表面与文本层级 | 可点击、可聚焦（视平台） |
| 按下 | 手指/指针按下可点条目 | 即时按压反馈 | 释放后触发导航或主操作；滑出过程中应避免误触主操作 |
| 悬浮 | 指针设备悬停（若平台支持） | 轻微表面或描边变化 | 不改变选中结果 |
| 获焦 | 键盘/遥控器等焦点落入 | 可见焦点环或等效指示 | 回车/确认触发与点击一致的主操作 |
| 选中 | 选择型列表中被选中 | 勾选、高亮或等效选中指示 | 单选互斥；多选可累加 |
| 激活/当前 | 分栏或主从布局中表示当前详情对应项 | 持续高亮，区别于瞬时按下 | 与「选中（批量）」语义分离，避免混用同一视觉 |
| 禁用 | 业务不可用 | 降低文本与图标强调 | 不响应点击与侧滑操作，不进入焦点顺序 |
| 加载/占位 | 数据未就绪 | 骨架或占位，不伪造最终文案 | 不提供会失败的主操作入口 |
整行点击绑定主任务。侧滑可露出删除、收藏、置顶等有限次要操作，图标表意须明确；操作区总宽度建议不超过行宽一半。行内常驻控件（如开关）与整行点击冲突时，应划分独立热区。
## 四、视觉规格
### 4.1、材质
| 属性 | Token / 规则 | 备注 |
| --- | --- | --- |
| 主文本 | `{color.text.primary}` | 一级信息 |
| 辅助文本 | `{color.text.secondary}` | 二级说明 |
| 元数据/弱化 | `{color.text.tertiary}` | 时间、次要属性 |
| 禁用文本/图标 | `{color.text.disabled}` | 与禁用态一致 |
| 按压/悬浮表面 | `{color.state.pressed}` 等 | 待接入 |
| 选中指示 | `{color.state.selected}` 或勾选控件色 | 与「当前项」区分 |
列表容器、侧滑操作底、媒体缩略图的表面与圆角跟随 HyperOS 表面与媒体材质语言；同列表内缩略图圆角应一致。
### 4.2、组件规格
效率型行高档位：48 / 56 / 64 / 72 / 96（单位待对齐 HyperOS）。内容型行高档位：64 / 72 / 80 / 96 / 120（单位待对齐 HyperOS）。
| 左侧标识类型 | 草案尺寸 | 说明 |
| --- | --- | --- |
| 状态点 Badge | 8×8 | 与图标角对齐规则待对齐 |
| 系统图标 | 16 / 24 | 效率型常用 |
| 头像 | 40 / 48 / 56 | 圆形或系统约定形状 |
| 1:1 预览 | 48 / 56 / 72 / 96 | 内容型常用 |
| 应用图标 | 48 / 56 / 64 | 按品牌规范裁切 |
| 宽图预览 | 最长边或高度档位约束 | 比例需在项目内定一种默认 |
右侧附属与主内容区间隔草案 12（单位待对齐）。组内垂直节奏、分组标题与首行距离、列表与页面边距使用间距 token。整行主操作热区高度不小于该行行高；行内独立控件与侧滑操作项具备独立可点热区。
## 五、多端设计指南
| 终端 | 规则 |
| --- | --- |
| 手机 | 单列；侧滑扩展次要操作；行高按效率/内容档选择 |
| 折叠屏 | 折叠单列，展开可分栏或增密；保持信息位对齐；主标题不被裁切失义 |
| 平板 | 可列表 + 详情分栏；补齐悬浮/获焦；当前项可见 |
| 手表等穿戴 | 本规范不展开；若需覆盖另补专项 |
较小宽度可隐藏次要元数据，但不可隐藏完成主任务所需的主标题。横滑条目带与「行侧滑操作」不是同一模式，若使用需另开说明。
